# WorkDeck 제품 개요

WorkDeck은 파일 관리와 작업 도구(터미널·SSH·FTP·미리보기)를 한 화면에 나란히 두는 크로스플랫폼 데스크톱 앱이다(macOS 우선 출시). 이 문서는 제품이 해결하는 문제, 타깃 사용자, 핵심 개념(사이드바·뷰·워크스페이스·콘텐츠 탭·분할·연결)을 소개하고, 기능 개발의 기준이 되는 문서 세트 전체의 목차를 안내한다.

## 1. 제품 개요

### 1.1 WorkDeck이란

WorkDeck은 "필요한 작업 도구를 한곳에 올려두는 작업용 데크"다. 파일 탐색, 로컬/SSH 터미널, SFTP/FTP 원격 파일 관리, 파일 미리보기를 하나의 창 안에서 콘텐츠 탭으로 열고 나란히 배치할 수 있다.

- **플랫폼**: Electron + TypeScript 기반 크로스플랫폼 데스크톱 앱. macOS를 우선 출시 대상으로 한다. (근거: [ADR-0001](../.forge/adr/0001-electron-over-tauri.md))
- **UI 모델**: VSCode식 구조. 화면 왼쪽의 사이드바에서 대상을 선택·탐색하고, 오른쪽의 워크스페이스에 타입별 콘텐츠 탭이 열린다.
- **차별점**: 파일 관리자·터미널·원격 접속 클라이언트를 별개 앱으로 오가는 대신, 하나의 통합 연결 프로필과 하나의 워크스페이스 위에서 모든 작업을 처리한다.
- **확장성**: 코어에 없는 기능은 확장 포인트에 끼우는 플러그인으로 추가한다(구현은 2차 마일스톤, [05-plugin-system.md](05-plugin-system.md)).

### 1.2 왜 필요한가

서버를 다루는 작업의 전형적인 흐름은 여러 앱에 흩어져 있다. 파일 위치는 파일 탐색기에서 확인하고, 명령 실행은 터미널 앱으로 전환하고, 원격 파일 전송은 별도의 FTP 클라이언트를 열고, 파일 내용 확인은 또 다른 편집기나 뷰어를 띄운다. 앱마다 접속 정보를 따로 관리하고, 같은 서버에 SSH 세션과 SFTP 세션을 각각 열어야 한다.

이 컨텍스트 전환 비용이 WorkDeck이 제거하려는 문제다:

- **앱 전환 비용** — 탐색기 ↔ 터미널 ↔ FTP 클라이언트 ↔ 뷰어를 오가며 흐름이 끊긴다.
- **접속 정보 중복** — 같은 서버의 SSH 계정과 FTP 계정을 앱마다 따로 등록·관리한다. WorkDeck에서는 하나의 연결 프로필을 "터미널로 열기"와 "파일로 열기" 두 방식으로 연다.
- **화면 배치 비용** — 로컬 파일과 원격 파일을 나란히 놓고 비교·전송하려면 창 두 개를 수동으로 배치해야 한다. WorkDeck에서는 워크스페이스 분할로 해결한다.

## 2. 타깃 사용자

서버를 직접 다루는 개발자·운영자가 핵심 타깃이다. 대표 페르소나 세 가지:

### 2.1 백엔드 개발자

여러 대의 개발·스테이징 서버에 수시로 SSH 접속해 로그를 확인하고 설정 파일을 열어 본다. 로컬에서 빌드한 산출물을 서버에 올리는 일도 잦다. 터미널 앱, 파일 탐색기, 전송 도구를 오가는 대신 연결 하나로 터미널과 원격 파일 목록을 같은 화면에 띄우고 싶어 한다.

### 2.2 서버 운영자 (인프라/DevOps)

관리 대상 서버가 수십 대다. 접속 정보 관리 자체가 일이며, 서버별로 자주 가는 경로(로그 디렉터리, 설정 디렉터리)가 정해져 있다. 연결 프로필 목록과 북마크로 "어느 서버의 어느 경로"에 한 번에 도달하고, 원격↔원격 파일 이동까지 같은 UX로 처리하고 싶어 한다.

### 2.3 웹 에이전시 실무자

고객사 호스팅 서버에 FTP/SFTP로 접속해 파일을 올리고 내리는 일이 업무의 큰 부분이다. 전송 전에 파일 내용(텍스트·이미지)을 빠르게 확인하고 싶지만, 기존 FTP 클라이언트의 미리보기는 빈약하다. 로컬 폴더와 원격 폴더를 나란히 놓고 드래그앤드롭으로 전송하는 익숙한 듀얼 패널 UX를 원한다.

## 3. 핵심 개념

WorkDeck의 화면과 동작은 아래 개념으로 설명된다. 정의는 프로젝트 용어집(`.forge/CONTEXT.md`)을 따르며, 모든 문서가 이 용어를 동일한 의미로 사용한다.

| 용어 | 정의 |
|------|------|
| **사이드바 (Sidebar)** | 화면 왼쪽의 고정 영역. 뷰를 전환하며 1차 선택·탐색을 담당한다. |
| **뷰 (View)** | 사이드바 안에서 전환되는 목록 화면. 파일·연결·북마크 세 가지가 있다. |
| **워크스페이스 (Workspace)** | 화면 오른쪽의 콘텐츠 영역. 콘텐츠 탭이 여러 개 동시에 열린다. |
| **콘텐츠 탭 (Content Tab)** | 워크스페이스에 열리는 작업 단위. 사이드바에서 항목을 활성화하면 타입에 맞는 탭이 열린다. |
| **분할 (Split)** | 워크스페이스를 좌우로 나눠 콘텐츠 탭 두 개를 나란히 배치하는 것. 듀얼 패널 파일 작업의 기초. |
| **연결 (Connection)** | 원격 서버 접속 정보를 담는 통합 프로필. "터미널로 열기"(SSH)와 "파일로 열기"(SFTP/FTP) 두 방식으로 열 수 있다. |
| **북마크 (Bookmark)** | 저장해 둔 위치(로컬 또는 원격 경로)의 바로가기. |
| **미리보기 (Preview)** | 파일 내용을 읽기 전용으로 보여주는 콘텐츠 탭. |
| **플러그인 (Plugin)** | WorkDeck에 기능을 추가하는 외부 설치 단위. 매니페스트로 자신이 쓰는 확장 포인트와 권한을 선언한다. |
| **확장 포인트 (Extension Point)** | 플러그인이 기능을 끼워 넣을 수 있도록 코어가 공식적으로 열어 둔 지점. 미리보기 렌더러, 연결 프로토콜, 콘텐츠 탭·사이드바 뷰, 명령·메뉴·테마 네 가지가 있다. |

### 3.1 핵심 상호작용 모델: 사이드바 → 콘텐츠 탭

WorkDeck의 모든 상호작용은 하나의 모델로 수렴한다. 사이드바의 뷰(파일·연결·북마크)에서 항목을 선택하면, 워크스페이스에 그 항목의 타입에 맞는 콘텐츠 탭이 열린다. 폴더를 선택하면 파일 목록 탭, 파일을 선택하면 미리보기 탭, 연결을 선택하면 SSH 터미널 탭 또는 원격 파일 목록 탭이 열린다. 같은 대상을 다시 선택하면 새 탭을 만들지 않고 기존 탭에 포커스한다.

```
사이드바에서 항목 선택
        ↓
   항목 타입 판별
        ├─ 폴더   → 파일 목록 탭 오픈
        ├─ 파일   → 미리보기 탭 오픈
        └─ 연결   → SSH 터미널 탭 또는 원격 파일 목록 탭 오픈
        ↓
같은 대상 재선택 시 → 기존 콘텐츠 탭에 포커스 (새 탭 생성 안 함)
```

이 모델 위에 분할이 더해지면 듀얼 패널 파일 작업이 된다. 워크스페이스를 좌우로 분할해 파일 목록 두 개를 나란히 놓으면, 로컬↔로컬·로컬↔원격·원격↔원격 어느 조합이든 같은 UX(F5 복사 / F6 이동 / 드래그앤드롭)로 파일을 옮길 수 있다.
