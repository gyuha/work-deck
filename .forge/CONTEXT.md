# WorkDeck

파일 관리와 작업 도구(터미널·SSH·FTP·미리보기)를 한 화면에 나란히 두는 데스크톱 앱. 이 문서는 프로젝트의 용어집이다.

## Language

**WorkDeck**:
이 제품의 공식 이름. README의 정의("필요한 작업 도구를 한곳에 올려두는 작업용 데크")를 따른다.
_Avoid_: TwinDeck (폐기된 후보명)

**사이드바 (Sidebar)**:
화면 왼쪽의 고정 영역. 뷰를 전환하며 1차 선택·탐색을 담당한다.

**뷰 (View)**:
사이드바 안에서 전환되는 목록 화면. 파일·연결·북마크 세 가지가 있다.
_Avoid_: 왼쪽 패널

**워크스페이스 (Workspace)**:
화면 오른쪽의 콘텐츠 영역. 콘텐츠 탭이 여러 개 동시에 열린다.
_Avoid_: 오른쪽 패널

**콘텐츠 탭 (Content Tab)**:
워크스페이스에 열리는 작업 단위. 사이드바에서 항목을 활성화하면 타입에 맞는 탭이 열린다(폴더→파일 목록, 파일→미리보기, 연결→터미널 또는 원격 파일 목록). 같은 대상을 다시 선택하면 기존 탭에 포커스한다. 예외: 로컬 터미널 탭은 사이드바 항목이 아니라 명령으로 생성되며, 이 중복 포커스 규칙의 대상이 아니다.

**분할 (Split)**:
워크스페이스를 좌우로 나눠 콘텐츠 탭 두 개를 나란히 배치하는 것. 두 파일 목록 간 복사/이동(듀얼 패널 파일 작업)의 기초가 된다.

**연결 (Connection)**:
원격 서버 접속 정보를 담는 통합 프로필. 하나의 연결을 "터미널로 열기"(SSH)와 "파일로 열기"(SFTP/FTP) 두 방식으로 열 수 있다.
_Avoid_: SSH 세션과 FTP 계정을 별개 개념으로 부르는 것

**북마크 (Bookmark)**:
저장해 둔 위치(로컬 또는 원격 경로)의 바로가기.

**미리보기 (Preview)**:
파일 내용을 읽기 전용으로 보여주는 콘텐츠 탭.

**플러그인 (Plugin)**:
WorkDeck에 기능을 추가하는 외부 설치 단위. 매니페스트로 자신이 쓰는 확장 포인트와 권한을 선언한다.
_Avoid_: 익스텐션, 애드온

**확장 포인트 (Extension Point)**:
플러그인이 기능을 끼워 넣을 수 있도록 코어가 공식적으로 열어 둔 지점. 미리보기 렌더러, 연결 프로토콜, 콘텐츠 탭·사이드바 뷰, 명령·메뉴·테마 네 가지가 있다.

**확장 호스트 (Extension Host)**:
플러그인 코드가 실행되는 별도 프로세스. 플러그인은 이 안에서 선언된 WorkDeck API로만 앱과 통신한다.

**매니페스트 (Manifest)**:
플러그인의 자기 선언 파일(plugin.json). 식별자·버전·진입점·사용하는 확장 포인트·요구 권한을 담는다.
